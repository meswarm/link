# Link — 智能工具中间件架构设计

> 设计日期：2026-04-10
> 状态：已批准

## 核心概念

Link 是一个 Agent 中间件框架。每个 Agent 实例：
- 拥有独立的 Matrix 账户，像真实用户一样出现在房间中
- 内置 LLM 决策引擎（Function Calling），理解用户自然语言并映射到工具操作
- 连接一组具体工具（API / CLI / 主动推送源）
- 作为独立 Python 进程运行

## 架构决策

- **方案 A：单体 Agent 框架** — 已选定
- 三层架构：Matrix 连接层 → LLM 决策层 → 工具适配层
- Python 技术栈，每个 Agent 独立进程
- YAML 配置驱动
- OpenAI 兼容 API（Qwen qwen3.6-plus）+ Function Calling

## 已确认的设计决策

| # | 问题 | 决策 |
|---|------|------|
| 1 | Matrix 加密 | MVP 不加密，后续可选 |
| 2 | 多房间 | ✅ 支持加入多个房间 |
| 3 | 用户权限 | 响应房间内所有消息，主要用户为本人 |
| 4 | 工具推送频率 | 低频（每天 5-8 次），无需限流 |
| 5 | 项目名 | `link` |

## 被拒绝的方案

**方案 B：分层服务架构** — 拒绝
- 理由：过度工程化，引入额外基础设施（Redis/ZMQ），不符合 YAGNI 原则
